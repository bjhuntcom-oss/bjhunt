# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest `main` (rolling) | Yes |
| Older releases | No — please upgrade |

## Reporting a Vulnerability

**Do NOT open public GitHub issues for security vulnerabilities.**

Report privately via:

1. **GitHub Security Advisories** (preferred): https://github.com/bjhuntcom-oss/bjhunt/security/advisories/new
2. **Email**: `security@bjhunt.com`

Please include:
- A description of the vulnerability
- Steps to reproduce
- Impact assessment

## Response SLA

| Stage | Target |
|---|---|
| Acknowledgement | Within 48 hours |
| Initial assessment | Within 5 business days |
| Fix for CRITICAL/HIGH | Within 30 days |
| Fix for MEDIUM | Within 90 days |
| Public disclosure | After fix shipped + 30 days grace period |

## Scope

In-scope:
- BJHUNT frontend (Next.js, Vercel)
- CI/CD workflows

Out-of-scope:
- Third-party dependencies (report upstream)
- Issues without production impact

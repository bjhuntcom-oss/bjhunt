## 1. CRITIQUES (à fixer en priorité)

### C-1 — `api_key_requests` n'a PAS de RLS
- **Source** : `pg_class` sur prod : `relrowsecurity=f, relforcerowsecurity=f`.
- **Risque** : table tenant-scoped (`org_id`) accessible cross-tenant via `bjhunt_app` une fois isolation DB activée.
- **Fix** : ajouter dans une nouvelle migration `0004_rls_api_key_requests.sql` :
  ```sql
  ALTER TABLE api_key_requests ENABLE ROW LEVEL SECURITY;
  ALTER TABLE api_key_requests FORCE ROW LEVEL SECURITY;
  CREATE POLICY api_key_requests_org_isolation ON api_key_requests
    USING (org_id = current_setting('app.current_org_id', true)::UUID)
    WITH CHECK (org_id = current_setting('app.current_org_id', true)::UUID);
  GRANT SELECT, INSERT, UPDATE, DELETE ON api_key_requests TO bjhunt_app;
  ```

### C-2 — `chat.ts` : 28 SQL via `adminSql` au lieu de `withOrg()`
- **Source** : `backend/src/routes/chat.ts:39, 161, 171, 176, 207, 243, 300, 371, 984` etc.
- **Impact** : actuellement non-bloquant car `adminSql` BYPASSRLS. Mais defense-in-depth DB-level absente. Toute fuite de validation app-level → cross-tenant leak.
- **Fix** : wrap chaque flow tenant-scoped en `withOrg(orgId, tx => …)`. Reste `adminSql` uniquement pour pre-session (login, ticket creation avant scope org).

### C-3 — VPS sshd : `PermitRootLogin yes` + `PasswordAuthentication yes`
- **Source** : `ssh bjhunt-vps "grep ... /etc/ssh/sshd_config"`.
- **Action TOI** : créer user `bjhunt` non-root + clé SSH, tester en session parallèle, puis :
  ```
  sed -i 's/^PermitRootLogin yes/PermitRootLogin no/; s/^PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
  systemctl reload sshd
  ```

### C-4 — `.env` VPS mode 644 (world-readable)
- **Source** : `stat -c %a /opt/bjhunt/app/.env` → `644`.
- **Fix** : `chmod 600 /opt/bjhunt/app/.env && chown root:root /opt/bjhunt/app/.env`. Container backend monte `.env` via compose, pas par chemin direct → safe.

### C-5 — Hostinger API token committed dans `.mcp.json:8`
- **Statut** : déjà flaggé (C-01). **Action TOI** : rotate via dashboard Hostinger, retirer la ligne.

## 1. CHAT SSE — bugs reproductibles

### CHAT-P0-1 — Course double-send / changement de conversation pendant streaming
- **Fichiers** : `app/[locale]/dashboard/chat/page.tsx:659, 906-919`

## 2. AUTH / 2FA

### AUTH-P1-1 — Email verification jamais enforcée
- **Fichiers** : `backend/src/routes/auth.ts:53-121` (register crée session direct), `schema.sql:32` (`email_verified` colonne existe mais zéro check)

### AUTH-P1-2 — Change password n'invalide pas autres sessions
- **Fichier** : `auth.ts:403-431`
- **Cause** : update `password_hash` seul. Cookies volés restent valides 30j.
- **Fix** : `deleteUserSessions(userId)` après update (sauf session courante).

### AUTH-P1-3 — Backup code 2FA consommation non atomique
- **Fichier** : `routes/two-factor.ts:227-233`
- **Cause** : verify+remove en deux étapes (SELECT puis UPDATE). Deux requêtes simultanées avec même backup code passent toutes les deux.
- **Fix** : `UPDATE … SET totp_backup_codes = array_remove(…) WHERE id=$1 AND $2 = ANY(totp_backup_codes) RETURNING …` atomique.

### AUTH-P1-4 — TempToken 2FA replayable
- **Fichiers** : `auth.ts:183-194`, `two-factor.ts:200-207`
- **Cause** : tempToken = `sha256(userId:secret:expiry):userId:expiry`. Pas de nonce, pas de tracking single-use. Brute-force TOTP unlimited tant que tempToken valide.

### AUTH-P1-5 — Lockout contournable via rotating IP
- **Fichier** : `auth/lockout.ts:39-42`
- **Cause** : key = `(email, ip)` sans cap absolu par email. 10 IPs × 5 essais = 50 essais/h sur même email.
- **Fix** : ajouter contre-clé `(email)` avec seuil plus large (ex 30/h).

### AUTH-P2-1 — CSRF accepte Origin manquant si session présente
- **Fichier** : `middleware/csrf.ts:33-44`
- **Note** : SameSite=Lax limite déjà l'attaque, mais durcir.

### AUTH-P2-2 — Logout = single session (pas multi-device)
- **Fichier** : `auth.ts:232-253` — pas d'option "logout all devices".

### AUTH-P2-3 — Reset email pas rate-limité par user

### AUTH-P2-4 — Backup code entropy ~47 bits
